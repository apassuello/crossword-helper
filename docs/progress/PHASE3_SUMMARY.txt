└──────────────────────────────────────────────────────────────────────────┘

Feature: Lock specific words in the grid that must be preserved during autofill

User Actions:
  • Right-click any cell → Toggles theme lock
  • Press Ctrl/Cmd+L   → Toggles theme lock (keyboard shortcut)
  • Locked cells       → Turn purple with lock icon 🔒
  • Start autofill     → Locked words stay unchanged

                             Purple border (#9c27b0)
                             Lock icon in corner

Autofill Panel shows: "Theme entries locked: 2"
                      "Right-click cells to lock/unlock theme words"


┌──────────────────────────────────────────────────────────────────────────┐
│ DATA FLOW (End-to-End)                                                   │
└──────────────────────────────────────────────────────────────────────────┘

1. USER INTERACTION (GridEditor.jsx)
   └─> Right-click cell → onToggleThemeLock(row, col)

2. STATE MANAGEMENT (App.jsx)
   └─> Updates grid cell: { ...cell, isThemeLocked: !isThemeLocked }

3. VISUAL FEEDBACK (GridEditor.jsx)
   └─> Purple background, border, lock icon render

4. EXTRACTION (AutofillPanel.jsx)
   └─> getThemeEntries() extracts complete words
       Output: {"(0,0,across)": "HELLO", "(0,0,down)": "HELP"}

5. API REQUEST (App.jsx)
   └─> fetch('/api/fill/with-progress', {
         body: { ..., theme_entries: {...} }
       })

6. VALIDATION (backend/api/validators.py)
   └─> Checks format: "(row,col,direction)" with word value

7. FILE CREATION (backend/api/routes.py)
   └─> Saves to temp file: /tmp/theme_abc123.json

8. CLI EXECUTION (cli/src/cli.py)
   └─> Loads JSON, converts format, passes to algorithms

9. AUTOFILL (hybrid_autofill.py, beam_search.py, iterative_repair.py)
   └─> Preserves theme entries, fills rest of grid

10. RESULT
    └─> Theme words unchanged ✓, other slots filled ✓
